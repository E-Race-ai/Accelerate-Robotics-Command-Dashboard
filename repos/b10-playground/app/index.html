B10 Patient Position Monitor application

Working Python BLE scanner + live web dashboard for the Minew B10. Captures sternum-mounted accelerometer data and infers patient body position (Supine, Prone, Lateral, Upright). Origin: LTC Monitor R&D project.

What it does

Runs a tight (1 s) macOS CoreBluetooth scan loop, parses the B10's proprietary 14-byte accel/gyro advertisement frame, classifies orientation against gravity, and accumulates time-in-position totals per day. Dashboard auto-opens at http://localhost:8765.

  • Real-time tilt, roll, head elevation
  • Live patient-figure visualization (rotates with the wearer)
  • Daily & weekly position-distribution charts
  • Frame history table (last 200 frames)
  • Daily JSON persistence for offline analysis

Setup

python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

macOS will prompt for Bluetooth permission on first run.

Run the dashboard

python b10_web.py

Browser opens automatically at http://localhost:8765. Position-time data accumulates per day under data/position_YYYY-MM-DD.json.

Position model

Sensor mounts on the sternum, Y axis pointing toward the head. The dashboard classifies orientation against the gravity vector.

Y = −1gStanding / Upright
Z = +1gSupine (on back)
Z = −1gProne (face down)
X = +1gLeft Lateral
X = −1gRight Lateral

Thresholds: 20° tilt = "tilted", 25° = full lateral turn, 45° = primary supine/prone/upright. Tunable in orientation_from_accel() in b10_web.py.

Reference hardware

SensorMinew B10 (IP66, 60-day battery, 3-axis accel + 3-axis gyro)
RadioBLE 5.0 — Eddystone + iBeacon + proprietary accel frame
Frame14 bytes, big-endian: A1 15 [ax ay az gx gy gz]
ScaleAccelerometer counts ÷ 256 = g
